The term is inspired from Data augmentation it is the process of artificially generating new data from existing data, primarily to train new machine learning (ML) models.

.
This leads to new dangers that threatnes software sustainabiliy, thus these concerns needs to be incorportaed in a new Agile framework: Augmented Scrum.

# The Problem

AI has changed software development.

It did not remove complexity. It moved it.

Instead of writing code, teams now spend more time understanding, validating, and controlling code generated by AI.

AI produces code faster than humans can fully understand it. Over time, this creates:

Loss of system understanding
Unclear or untraceable decisions
Unpredictable system behavior

This is not just technical debt. It is cognitive debt.

When cognitive debt grows too large, systems eventually fail.

# The Idea

Augmented Scrum is a lightweight extension of Scrum for AI-assisted development.

Humans remain responsible for design and control. AI is used only as an execution accelerator within clear limits.

The main addition is the concept of entropy control.

# Core Insight

AI increases output, but not understanding.

Without control:

More code does not mean more progress
Faster delivery can delay failure instead of preventing it

# What Is Entropy?

In this context, entropy means:

Increasing unpredictability
Loss of structure
Loss of traceability

AI can increase entropy if not managed properly.

If left unchecked, systems become:

Hard to understand
Risky to change
Expensive to maintain

# Key Concepts
Definition of Control

An increment is valid only if:

It is understandable
Its behavior is predictable
It can be safely modified

Done does not mean controlled.

# Cognitive Budget

Each sprint has a limit on how much AI-generated change the team can fully understand.

Exceeding this limit creates invisible debt.

# AI Is Not a Team Member

AI does not:

Take responsibility
Understand context
Retain memory

AI increases output, but not accountability.

# The Control Loop

Human design
→ Bounded AI generation
→ Human validation
→ Control check
→ Entropy reduction
→ Repeat

# What Happens Without Control

AI-driven systems typically fail in stages:

First: increased speed
Then: growing confusion
Then: instability
Finally: system collapse

The failure is usually sudden, not gradual.

# Purpose

Augmented Scrum ensures that:

What teams gain in speed, they do not lose in control.
